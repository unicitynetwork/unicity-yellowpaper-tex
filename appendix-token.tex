\chapter{Unicity Token Encoding}\label{app:execution-layer}

This appendix specifies the concrete encodings for the Unicity Token defined in the Execution Layer chapter. All structures are serialized using CBOR (Appendix~\ref{app:encodings}) with deterministic encoding rules.

\section{Constants}\label{app:el-constants}

\begin{itemize}
	\item $\mathsf{MINT\_SUFFIX} = \mathsf{SHA\text{-}256}(\texttt{"TOKENID"})$ -- a fixed 32-byte domain separator used to derive the source state hash of mint transactions.
	\item $\mathsf{MINTER\_SECRET}$ = \\
	  \hspace*{10mm}\texttt{0x495f414d5f554e4956455253414c5f4d494e5445525f464f525f} \\
	  \hspace*{10mm} (ASCII: \texttt{"I\_AM\_UNIVERSAL\_MINTER\_FOR\_"}) \\
	  -- a fixed 26-byte constant used to derive deterministic minting key pairs.
	\item The minting key pair for a token with identifier $\mathsf{id}$ is derived as:
	\[
	\mathsf{sk}_\mathsf{mint} = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\mathsf{MINTER\_SECRET}, \mathsf{id}))
	\]
	The corresponding public key $\mathsf{pk}_\mathsf{mint}$ is the secp256k1 public key for $\mathsf{sk}_\mathsf{mint}$. The private key is intentionally public; the key pair exists solely to provide a uniform request interface with the Unicity Service.
\end{itemize}

\section{Predicate Encoding}\label{app:predicate-encoding}

A predicate is a value $\predi$ whose authority over a token state is defined by its canonical byte encoding
\[
\encpred(\predi) \in \{0,1\}^* .
\]

Predicate equality means byte equality of canonical encodings:
\[
\predi_1 = \predi_2 \quad\Longleftrightarrow\quad
\encpred(\predi_1) = \encpred(\predi_2).
\]

The encoding $\encpred$ is implemented as encoding to a CBOR array with three top-level elements:

\begin{longtable}{clp{8cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & unsigned integer & Predicate engine identifier. Value $1$ denotes the built-in engine. \\
1 & byte string & Predicate type code within the engine. \\
2 & byte string or CBOR item & Serialized parameters (encoding depends on the predicate type; see below). \\
\end{longtable}

\noindent Throughout this section, $\mathsf{pk}$ denotes a 33-byte compressed secp256k1 public key (see Sec.~\ref{app:signature-format}), and timestamps $\tau \in \mathbb{T}$ are encoded as CBOR unsigned integers (seconds; same convention as $C^\mathsf{r}.t_r$). Hashes are 32 bytes.

\subsection{Built-in Predicate Type Codes}

\begin{longtable}{llp{10cm}}
\textbf{Code} & \textbf{Predicate} & \textbf{Parameters encoding} \\
\hline
\endhead
\texttt{0x01} & $\predi_\mathsf{sig}$ (Signature) & Recipient public key $\mathsf{pk}$. \\
\texttt{0x02} & $\predi_\mathsf{burn}$ (Burn) & reason hash $h_\mathsf{reason}$. \\
\texttt{0x03} & $\predi_\mathsf{tlock}$ (Timelock) & CBOR array[2]: $[\mathsf{pk}\ (33\text{-byte bstr}),\ \tau_\mathsf{min}\ (\text{uint})]$. \\
\texttt{0x04} & $\predi_\mathsf{p2pkh}$ (Pay-to-Public-Key-Hash) & public key hash $h_\mathsf{pk}$. \\
\texttt{0x05} & $\predi_\mathsf{p2sh}$ (Pay-to-Script-Hash) & predicate hash $h_\predi = H(\mathsf{CBOR}(\predi'))$. \\
\texttt{0x06} & $\predi_\mathsf{msig}$ (Multi-signature) & CBOR array of $n$ entries of $\mathsf{pk}_i$. \\
\texttt{0x07} & $\predi_\mathsf{tsig}$ (Threshold-signature) & CBOR array[2]: $[k\ (\text{uint}),\ [\mathsf{pk}_1,\ldots,\mathsf{pk}_n]]$; $1 \le k \le n$. \\
\texttt{0x08} & $\predi_\mathsf{htlc}$ (Hashed Timelock) & CBOR array[4]: $[\mathsf{pk},\ \mathsf{pk}',\ y\ (32\text{-byte bstr}),\ \tau_\mathsf{max}\ (\text{uint})]$. \\
\end{longtable}

\subsection{Unlocking Argument Encoding}

The unlocking argument $u$ accompanying a Unicity Service Request (Sec.~\ref{sec:request-validation}) is encoded according to the current owner's predicate type. Signatures are 65 bytes in the $R\|S\|V$ format defined in Sec.~\ref{app:signature-format}.

\begin{longtable}{cp{10cm}}
\textbf{Pred.} & \textbf{Unlocking argument encoding} \\
\hline
\endhead
$\predi_\mathsf{sig}$    & signature $s$. \\
$\predi_\mathsf{burn}$   & None; the predicate never evaluates to $1$ and no unlocking argument exists. \\
$\predi_\mathsf{tlock}$  & signature $s$. \\
$\predi_\mathsf{p2pkh}$  & CBOR array[2]: $[\mathsf{pk},\ s]$. \\
$\predi_\mathsf{p2sh}$   & CBOR array[2]: $[\mathsf{CBOR}(\predi')\ (\text{bstr}),\ u'\ (\text{bstr})]$, where $\predi'$ is the revealed predicate and $u'$ is the unlocking argument encoded according to $\predi'$'s type. \\
$\predi_\mathsf{msig}$   & CBOR array of $n$ signatures $[s_1, \ldots, s_n]$, in the same order as the parameter public keys. \\
$\predi_\mathsf{tsig}$   & CBOR array of $k$ entries, each a CBOR array[2]: $[s_i,\ j_i\ (\text{uint})]$, where $j_i \in \{1,\ldots,n\}$ is the index of the parameter public key used to produce $s_i$. Indices must be pairwise distinct. \\
$\predi_\mathsf{htlc}$   & CBOR array[2]: $[x\ (\text{bstr}),\ s\ (65\text{-byte bstr})]$. For the claim path $x$ is the preimage with $H(x) = y$; for the refund path $x$ may be empty ($\mathtt{h'}$). \\
\end{longtable}


\subsection{Predicate Soundness}\label{app:predicate-soundness}

The following rules must apply to all predicates, including user-defined, custom programmable predicates.


\begin{enumerate}
  \item \textbf{Canonical encoding.}
    $\encpred$ MUST be deterministic, injective on valid predicates, and encoded using deterministic CBOR. A conforming decoder MUST reject non-canonical CBOR encodings, trailing bytes, duplicate map keys, invalid field counts, invalid field types, invalid public keys, invalid thresholds, and any type-specific malformed parameters.

  \item \textbf{Round-trip stability.}
    For every accepted predicate byte string $b$:
    \[
      \encpred(\decpred(b)) = b.
    \]
    For every valid predicate object $\predi$, repeated evaluations of $\encpred(\predi)$ MUST return identical bytes.

  \item \textbf{Predicate immutability.}
    The value used in defined protocol steps is the canonical byte string $\encpred(\predi)$, not an in-memory object or function. Implementations MUST store and compare this byte string.

    \item \textbf{State identifier binding.}
    The state identifier MUST be derived from exactly the same canonical predicate bytes that are authenticated by the unlocking argument:
    \[
      \mathsf{sid} =
      \mathsf{SHA\text{-}256}(
        \mathsf{CBOR}(\encpred(\predi), \sthash)
      ).
    \]
    Verification of an inclusion proof for a transaction spending state $(\predi,\sthash)$ MUST reject unless the predicate and source state
  hash contained in the certified request are byte-identical to the transaction's current-owner predicate and source state hash.

    \item \textbf{Deterministic verification relation.}
    Each predicate type defines a deterministic verification relation
    \[
      \mathsf{VerifyPred}(\predi, u, \sthash, \txhash, C) \in \{0,1\},
    \]
    where $u$ is the unlocking argument and $C$ is explicit public verification context, such as a certified timestamp if required. For fixed inputs, the result MUST be stable and MUST NOT depend on hidden local state, network calls, randomness, or implementation-specific
  object identity.



    \item \textbf{Unlocking Argument canonicality.}
    Each unlocking argument $u$ MUST have a deterministic CBOR encoding and a predicate-type-specific decoder. The decoder MUST reject malformed, non-canonical, ambiguous, duplicate, out-of-range, or trailing-byte encodings. Re-encoding an accepted unlocking argument MUST reproduce the accepted byte string.

    \item \textbf{No side effects by unlock argument.}
    The unlocking argument MAY provide witnesses required by the predicate, such as signatures, public keys for P2PKH, scripts for P2SH, or hash preimages. It MUST NOT alter the current-owner predicate, source state hash, transaction hash, or state identifier. If $u$ reveals a hidden predicate $\predi'$ for P2SH, verification MUST first check:
    \[
      \mathsf{SHA\text{-}256}(\encpred(\predi')) = h_\predi
    \]
    before even trying to parse $\predi'$.

    \item \textbf{Explicit public context.}
    Predicates that depend on time, block height, round number, or certificate metadata MUST define precisely which certified public field is used. Such predicates MUST NOT depend on any external data not committed by spending Unicity Certificate.

    \item \textbf{User-defined predicate safety.}
    Extension engines MUST define:
    \[
      (\mathsf{engine},\mathsf{code},\mathsf{params},\mathsf{unlock},\mathsf{VerifyPred})
    \]
    with canonical encodings for both parameters and unlocking arguments. Extension predicates MUST satisfy the same determinism,
  immutability, and round-trip requirements as built-in predicates.
  \end{enumerate}

  \noindent These requirements ensure that a relying party, wallet, SDK, and Unicity Service all derive the same state identifier and verify the same authorization statement.

\subsection{Recipient Field}

The \emph{recipient} field in both mint transactions and transactions stores the next owner's predicate $\predi'$ encoded as in Sec.~\ref{app:predicate-encoding}.

By convention, a mint transaction uses $\predi' = \predi_\mathsf{sig}(\mathsf{pk'})$ (type code \texttt{0x01}) for the first owner.

If a sender wishes to commit only to a hash of a complex predicate (hiding its structure until spend time), they can do so explicitly by setting $\predi' = \predi_\mathsf{p2sh}(H(\mathsf{CBOR}(\predi'')))$ (Sec.~\ref{sec:p2sh}); the P2SH predicate is itself encoded as a first-class predicate in the recipient field.


\section{Transaction Encoding}\label{app:transaction-encoding}

\subsection{Mint Transaction}

A mint transaction is encoded as a CBOR array with three elements:

\begin{longtable}{clp{10cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & byte string & Recipient: CBOR-encoded first owner's predicate $\mathsf{CBOR}(\predi')$ (Sec.~\ref{app:predicate-encoding}); by convention $\predi' = \predi_\mathsf{sig}(\mathsf{pk'})$ \\
1 & byte string & Token Identifier $\mathsf{id}$ (32 bytes) \\
2 & byte string & Token Type $\mathsf{ty}$ (32 bytes) \\
3 & byte string & Mint Reason $\mathsf{reason}$ (optional, arbitrary length) \\
4 & byte string & Auxiliary Payload $\auxd'$ (optional, arbitrary length) \\
\end{longtable}


\subsection{Transfer Transaction}

A transfer transaction (spending a mint or prior transaction output) is encoded as a CBOR array with five elements:

\begin{longtable}{clp{10cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & byte string & Source state hash $\sthash$ (32 bytes) \\
1 & byte string & Unlocking Argument: CBOR-encoded $\mathsf{CBOR}(u)$ (Sec.~\ref{app:predicate-encoding}); must unlock the recipient predicate of the preceding transaction \\
2 & byte string & Recipient: CBOR-encoded next owner's predicate $\mathsf{CBOR}(\predi')$ (Sec.~\ref{app:predicate-encoding}) \\
3 & byte string & Random value $x$ (32 bytes) \\
4 & byte string & Auxiliary data $\auxd'$ \\
\end{longtable}


\subsection{Certified Transaction}

A certified transaction (mint or transfer) is encoded as a CBOR array with two elements:

\begin{longtable}{clp{8cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & CBOR array & The transaction \\
1 & CBOR structure & Inclusion proof $\pi$ \\
\end{longtable}


\subsection{Token}

A token is encoded as a CBOR array with two elements:

\begin{longtable}{clp{8cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & CBOR array & Certified mint transaction \\
1 & CBOR array & Array of certified transactions (may be empty) \\
\end{longtable}


\section{Hash Derivations}\label{app:hash-derivations}

All hash derivations use SHA-256 over CBOR-encoded inputs.

\subsection{State Identifier}

The state identifier is derived from the predicate and state hash:
\[
\mathsf{sid} = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\predi, \sthash))
\]

\subsection{State Hash}

For mint transactions:
\[
\sthash = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\mathsf{id}, \mathsf{MINT\_SUFFIX}))
\]

For transfer transactions, the next state hash:
\[
\sthash' = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\sthash, x))
\]

\subsection{Transaction Hash} % the leaf's value of aggregator tree

For mint transactions:
\[
\txhash = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\predi', \mathsf{id}, [\mathsf{ty}, \mathsf{reason}, \auxd']))
\]

For transactions:
\[
\txhash = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\predi', x, \auxd'))
\]

where $\mathsf{CBOR}(\ldots)$ denotes the CBOR array encoding of the arguments as specified in Sec.~\ref{app:transaction-encoding}.


\section{Signature Format}\label{app:signature-format}

All standard predicates that verify digital signatures ($\predi_\mathsf{sig}$, $\predi_\mathsf{p2pkh}$, $\predi_\mathsf{tlock}$, $\predi_\mathsf{msig}$, $\predi_\mathsf{tsig}$, $\predi_\mathsf{htlc}$) use ECDSA over the secp256k1 curve with the following formats.

\begin{itemize}
	\item \textbf{Public keys}: compressed format, 33 bytes (1-byte prefix $\texttt{0x02}$ or $\texttt{0x03}$, followed by 32-byte $x$-coordinate).
	\item \textbf{Signatures}: 65 bytes ($R \mathbin{\|} S \mathbin{\|} V$), where $R$ and $S$ are 32-byte integers and $V$ is a 1-byte recovery identifier.
	\item \textbf{Signed message}: the predicate message hash $m = \mathsf{SHA\text{-}256}(\mathsf{CBOR}(\sthash, \txhash))$, as defined in Sec.~\ref{sec:predicate-framework}.
\end{itemize}


\section{Split Proof Encoding}\label{app:split-proof-encoding}

\subsection{Split Reason}

A split reason is encoded as a CBOR array with two elements:

\begin{longtable}{clp{8cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & CBOR structure & The burned source token (complete token encoding) \\
1 & CBOR array & Array of split reason proofs (one per asset type) \\
\end{longtable}

\subsection{Split Reason Proof}

A split reason proof is encoded as a CBOR array with three elements:

\begin{longtable}{clp{8cm}}
\textbf{Index} & \textbf{Type} & \textbf{Description} \\
\hline
0 & byte string & Asset identifier $\mathsf{aid}$ \\
1 & CBOR structure & Aggregation tree path (Sparse Merkle Tree path to the aggregation root) \\
2 & CBOR structure & Asset sum tree path (Sparse Merkle Sum Tree path proving value allocation) \\
\end{longtable}
